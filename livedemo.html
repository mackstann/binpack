This is a demonstration of a 2D bin packing algorithm, operating on a continuously randomly-generated set of rectangles. Credit for this algorithm goes to Jim Scott. Read his page to learn how it works.

I don't claim this to be any sort of optimal implementation. Ideally, you would have a pre-generated set of rectangles you want to pack into a certain area. You could then pre-sort them in different ways to find more efficient packing arrangements. There are also completely different algorithms for accomplishing this task. This is merely one. Bin packing is an NP-hard problem, meaning that the only way to find the best result is to try every possible combination in a brute force manner, which becomes computationally impractical or impossible when you have many items to pack together.
